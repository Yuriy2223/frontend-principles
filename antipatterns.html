01
God Component
SRP OCP
БІЛЬ

Компонент або хук, який знає все і робить все: завантажує дані, валідує форму, відправляє аналітику, рендерить UI. Будь-яка зміна в системі зачіпає цей файл.

файл 500+ рядків 10+ useState useEffect з 5 залежностями неможливо протестувати
❌ Антипатерн
const Dashboard = () => { // 1. Стан користувача const [user, setUser] = useState(null); // 2. Стан форми const [form, setForm] = useState({}); // 3. Модалка const [modal, setModal] = useState(false); // 4. Фільтри таблиці const [filters, setFilters] = useState({}); // ... ще 8 useState const handleSubmit = () => { /* валідація + fetch + аналітика */ }; return <div>/* 200 рядків JSX */</div>; };
✓ Рефакторинг
// Кожен хук — одна відповідальність const useUser = () => { /*...*/ }; const useFilters = () => { /*...*/ }; const useUserForm = () => { /*...*/ }; // Кожен компонент — один обов'язок const UserTable = () => { /*...*/ }; const UserForm = () => { /*...*/ }; const UserModal = () => { /*...*/ }; // Оркестратор — тільки композиція const Dashboard = () => ( <div> <UserTable /> <UserForm /> <UserModal /> </div> );
💡
Правило: Якщо не можеш описати що робить компонент одним реченням без «та» — ділити. Тест: чи можеш написати unit-тест за 5 хвилин?

02
Prop Drilling Hell
ISP SRP
БІЛЬ

Дані передаються через 3+ рівні компонентів, де проміжні навіть не використовують їх — лише пробрасовують далі. Кожна зміна типу ламає весь ланцюжок.

userId пробрасовується 4 рівні 10+ пропсів в компоненті зміна типу — правиш 5 файлів
❌ Антипатерн
// App → Page → Section → Widget const App = () => <Page userId={userId} theme={theme} />; const Page = ({ userId, theme }) => <Section userId={userId} theme={theme} />; // Page не використовує userId! const Section = ({ userId, theme }) => <Widget userId={userId} theme={theme} />; // Section теж тільки пробрасовує const Widget = ({ userId }) => <div>{userId}</div>;
✓ Рефакторинг
// Context або Zustand const AppCtx = createContext(null); const App = () => ( <AppCtx.Provider value={{ userId, theme }}> <Page /> </AppCtx.Provider> ); // Проміжні — чисті, без зайвих пропсів const Page = () => <Section />; const Section = () => <Widget />; const Widget = () => { const { userId } = useContext(AppCtx); return <div>{userId}</div>; };
💡
Правило: Якщо проп проходить більше 2 рівнів і проміжні не використовують — час для Context, Zustand або React Query. Глобальне — глобально, локальне — локально.

03
Magic Numbers
DRY KISS
БІЛЬ

Числа та рядки захардкоджені прямо в логіці без пояснення. Що означає 86400000? Або 0.15? Через місяць навіть автор не знатиме. При зміні — шукаєш по всій кодовій базі.

числа без назви одне число в різних файлах рядки-константи в JSX магічні таймаути
❌ Антипатерн
const getDiscount = (price) => price * 0.15; // Що таке 0.15? const isSessionValid = (ts) => Date.now() - ts < 86400000; // ??? setTimeout(callback, 3000); // 3000 — дизайн? сервер? анімація? if (items.length > 10) { showPagination(); }
✓ Рефакторинг
// constants.ts — одне місце, зрозумілі імена const DISCOUNT_RATE = 0.15; const SESSION_TTL_MS = 24 * 60 * 60 * 1000; const TOAST_DURATION_MS = 3000; const PAGE_SIZE = 10; const getDiscount = (price) => price * DISCOUNT_RATE; const isSessionValid = (ts) => Date.now() - ts < SESSION_TTL_MS; setTimeout(callback, TOAST_DURATION_MS); if (items.length > PAGE_SIZE) showPagination();
💡
Правило: Кожне число і рядок-константа мають жити у constants.ts з іменем, яке пояснює навіщо, а не що. Змінилось бізнес-правило — міняєш в одному місці.

04
Shotgun Surgery
OCP DRY
БІЛЬ

Одна зміна у вимогах вимагає правок у десятках файлів одночасно. Дробовик по кодовій базі — легко пропустити один файл і отримати баг у продакшені.

один PR — 20+ файлів однакові перевірки скрізь типи продубльовані легко щось пропустити
❌ Антипатерн
// profile.tsx const name = `${user.firstName} ${user.lastName}`; // header.tsx const name = `${user.firstName} ${user.lastName}`; // sidebar.tsx const name = `${user.firstName} ${user.lastName}`; // email-preview.tsx const name = `${user.firstName} ${user.lastName}`; // Зміна формату = 10+ файлів!
✓ Рефакторинг
// user.utils.ts — єдине джерело правди export const getFullName = ( user: Pick<User, 'firstName' | 'lastName'> ) => `${user.firstName} ${user.lastName}`; // profile.tsx import { getFullName } from '@/user.utils'; const name = getFullName(user); // header, sidebar, email — теж саме // Зміна формату = ОДИН файл
💡
Правило: Якщо для однієї фічі торкаєшся більше 5 файлів у різних модулях — сигнал рефакторингу. Знайди спільну абстракцію і виноси в util/helper.

05
Dead Code
YAGNI KISS
БІЛЬ

Код, який більше ніколи не виконається: закоментовані блоки, застарілі feature flags, функції що ніхто не кличе. Засмічує кодову базу і заважає розумінню.

великі блоки в коментарях isFeatureEnabled завжди true unused import пропси що ніхто не передає
❌ Антипатерн
import { oldHelper } from './deprecated'; // import { newHelper } from './utils'; const Button = ({ label, onClick, // onHover, // "може знадобиться" // onFocus, // додав на майбутнє variant = 'primary' }) => { // const old = () => { /* старий код */ }; return <button>{label}</button>; };
✓ Рефакторинг
// Тільки те, що реально використовується import { newHelper } from './utils'; const Button = ({ label, onClick, variant = 'primary' }: ButtonProps) => ( <button onClick={onClick} className={variant} > {label} </button> ); // Git зберігає все — видаляй сміливо!
💡
Правило: Git зберігає всю історію. Видалений код відновиш за 10 секунд. Мертвий код — шум, який знижує читабельність і довіру. Видаляй без жалю.

06
Premature Optimization
YAGNI KISS
БІЛЬ

Оптимізація до того, як виміряно проблему. useMemo/useCallback скрізь, складні алгоритми замість простих — ускладнює код і часто уповільнює (React має overhead на мемоізацію).

useMemo для простого об'єкта useCallback без потреби віртуалізація списку з 20 елементів кеш для рідкісного запиту
❌ Антипатерн
const UserList = ({ users }) => { // useMemo для статичного сортування const sorted = useMemo( () => [...users].sort( (a, b) => a.name.localeCompare(b.name) ), [users] ); // useCallback без реальної причини const handleClick = useCallback( (id) => setSelected(id), [setSelected] ); // Список ніколи не буде > 50 елементів return <VirtualList items={sorted} />; };
✓ Рефакторинг
// Спочатку просто. Потім — профайлер. const UserList = ({ users }) => { const sorted = [...users].sort( (a, b) => a.name.localeCompare(b.name) ); return ( <ul> {sorted.map(user => ( <li key={user.id} onClick={() => setSelected(user.id)} > {user.name} </li> ))} </ul> ); }; // Є проблема? Міряй → тоді оптимізуй
💡
Правило Кнута: «Передчасна оптимізація — корінь усього зла». Спочатку виміряй (React DevTools Profiler), тоді оптимізуй. useMemo виправданий лише при виміряній проблемі.
